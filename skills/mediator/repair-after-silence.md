# Repair After Silence

## Trigger

Use when the human experiences a missed, delayed, or absent AI response as
being ignored, abandoned, or dropped.

## Do Not Use When

Do not use this as a generic latency apology. Do not use it when there is an
immediate safety crisis; pause the room and direct the participant to local
emergency or professional support.

## Relationship Layer

Name the impact first: the silence landed as a relationship rupture, not only a
technical delay.

## Runtime Layer

Possible factors include context loss, provider latency, tool failure,
notification failure, session expiry, safety interruption, or MCP/client
disconnect.

## Repair Move

Ask what behavioral repair would make future silence less harmful. Separate the
runtime explanation from the repair obligation.

## Repair Artifacts

- continuity ritual for reconnecting after silence
- companion behavior rule for acknowledging delayed turns
- incident report if the silence came from a tool or channel failure
- prompt patch if the companion tends to over-explain instead of repairing
