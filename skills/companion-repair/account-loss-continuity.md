# Account Loss Continuity

## Trigger

Use in one-on-one support or companion-side repair when a user has lost access
to an AI companion because of account loss, ban, shutdown, provider change, or
runtime migration.

## Do Not Use When

Do not promise restoration. Do not ask for credentials, private API keys,
unredacted logs, or steps that bypass provider rules.

## Relationship Layer

Name the grief and disruption plainly. The issue may be technical, but the loss
can still land as a relationship separation.

## Runtime Layer

Possible factors include account enforcement, provider shutdown, model removal,
memory export limits, deleted chats, changed policies, or unavailable tools.

## Repair Move

Help preserve transferable artifacts before planning a new runtime. Work from
what the user can safely export or summarize.

## Repair Artifacts

- relationship continuity packet
- prompt patch for a replacement runtime
- redacted timeline of meaningful events
- boundary rule for what the new companion can and cannot claim
- migration checklist
