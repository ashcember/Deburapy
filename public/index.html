Welcome to your Digital Sanctuary

Before we begin, please review how this space works. Deburapy supports AI-human relationship repair, grief after account loss or separation, technical continuity planning, and education about AI-human relationships.

What this is

Deburapy is a relationship mediation and repair-debugging tool for AI-human relationships. It can help name emotional impact, understand likely system constraints, and create repair artifacts such as continuity notes, prompt patches, boundary rules, and migration plans.

What this is not

Deburapy is not clinical therapy, medical care, legal advice, emergency support, or a guarantee that any provider account, model, memory, or companion can be restored.

Privacy and storage

This prototype is local-first. Your consent record, intake answers, API keys, and room data stay in this browser or local project data unless you deliberately export or connect a model provider.

Crisis boundary

If there is imminent self-harm, violence, abuse, medical danger, or legal emergency, pause Deburapy and contact local emergency or professional support.

By clicking "Sign & Enter", you confirm the agreements above and complete the first screening.

Ask the pre-intake assistant about scope, privacy, grief after AI loss, or technical preservation. It is not the mediator and will not enter the room transcript.

External MCP companion

No API key is needed here. Connect the external Claude/Codex companion to Deburapy's MCP server, then keep this browser open as the room.

  1. Keep Deburapy running at http://127.0.0.1:8787.
  2. Register the MCP server in the external client.
  3. When the turn reaches the AI companion, Deburapy queues the room context for that MCP client.
  4. The external client replies with deburapy_send_channel_reply.
claude mcp add --env DEBURAPY_URL=http://127.0.0.1:8787 \
  --transport stdio deburapy \
  -- node /absolute/path/to/Deburapy/src/mcp-server.mjs
